# DraftCode 选秀作战室 · 前端体验设计（design.md）

> 目标：把"NBA 选秀预测 Agent 系统"做成一个**渐进式、电影感、可交互**的单页体验——
> 既适合**现场演示**、也适合**录一条 ≤60s 的 demo 视频**，并能一步步讲清系统全流程。
> **拒绝**把结果平铺在一张静态长页上。

---

## 1. 设计世界观

赛博 / EVA 终端 · "选秀作战室"。延续本机 `local_nbadraft_workspace/设计元素/` 与路演 PPT（`index.src.html`）已确立的视觉语言：

- **绿黑数据终端**：纯黑底 + 矩阵数字雨 + 扫描线 + 颗粒 + 细网格。
- **EVA 故障明朝体字标** + **像素字**（FusionPixel12）数据/状态 + **得意黑**（DeyiHei）展示文字。
- **玻璃控制台**面板、章节冲击、白闪震颤转场。
- Persona 5 式**尖锐斜切几何 / 冲击动势**作为交互点缀。

参考 `02_视觉元素/`（暗色发光展示、玻璃控制台、章节冲击页、流程路线、交互原型嵌入）与
`03_动效资产/`（FLIP 连续转场、内容级联入场、滚动触发、章节炸场白闪震颤、WebGL 噪点首屏、CoverFlow）。

## 2. 色彩 / 字体 / 材质（沿用 `01_设计规范/03_色彩字体材质.md`）

| 角色 | 值 | 职责 |
|--|--|--|
| 主背景 | `#000402` / `#020805` | 终端黑 |
| 主文字 | `#f1fff7` | 正文/标题 |
| 次文字 | `#9dc6b1` | 说明 |
| **主强调 绿** | `#37ff8b` / `#00ff66` | **仅**系统激活 / 置信通过 / 数据锁定 / 路径 |
| 辅 琥珀 | `#f1c96b` | 资金信号 / 警示 / 锁定标记 |
| 辅 青 | `#62f2ff` | 数据流 / 链接 |
| 危险 红 | `#ff4473` | 红队 / 背离 / 风险 |

- **Display = 得意黑**（标题、正文、强调数字；缺字回退 PingFang，合规）。
- **Pixel = FusionPixel12**（状态、编号、日志、百分比、终端——不用于长句）。
- **字标 = 系统明朝体**（Songti/STSong/Hiragino Mincho）+ EVA RGB 错位故障辉光。
- 字体全部 **base64 内嵌**，页面自包含；头像与雷达也内嵌 → 单文件可离线打开。
- **绿色不当墙纸**：不把每个边框/标题/卡片都刷绿（规范红线）。

## 3. 体验流程 —— 六幕渐进式（核心）

整页是一个**场景状态机**（不是滚动长页）。每幕：**有动画、有分析、有交互**，然后动画过渡到下一幕。

### 幕 0 · BOOT 封面（大页面，点击进入）
- 全屏 EVA 封面：密集矩阵雨 + 故障字标「选秀作战室 / DRAFTCODE」+ 竖排繁体「作戰室」。
- 中央一个**呼吸态大按钮**：`▶ 进入作战室 · ENTER WAR ROOM`。
- 点击 → **白闪 + 故障 + 推进镜头**（缩放/擦除）转场 → 幕 1。

### 幕 1 · 作战室上线（WAR ROOM ONLINE）
- 终端**逐行打字**启动日志：`booting commissioner… loading 124 prospects… 30 GM agents online… signals: talent/market/money`。
- 日志走完 → HUD 外壳显影（顶部状态条 + 三信号指示灯 + 左侧步骤导航）。
- 自动或点击 `开始推演` → 幕 2。

### 幕 2 · STEP 01 天赋（TALENT BIG BOARD）
- 124 名 prospect 以**级联/粒子**飞入，按天赋分聚成 Big Board。
- 分析卡（玻璃面板）：速率指标为脊（REB%/AST%/BLK%）、身体 Z-score、年龄修正。
- 交互：悬停球员 → 浮出天赋分；可拖动/筛选。→ 下一步。

### 幕 3 · STEP 02 市场（专家 + 资金 · 两轴背离）
- 左：专家 mock 共识加权；右：博彩赔率 → 隐含 → **de-vig 去水**。两股信号流入。
- **背离火花**：天赋×市场、专家×资金不一致处迸发红/琥珀脉冲，触发 `gpt-5.5 裁决` 动画。
- 分析卡：分歧就是 alpha，钱是最快的情报。交互：切换两轴、点背离看裁决。→ 下一步。

### 幕 4 · STEP 03 融合 + 蒙特卡洛（FUSION + MONTE CARLO）
- 三股信号流**汇合**成一条主路径（沿用 PPT 的流程路线 + 流光 dash）。
- **蒙特卡洛 1500 次抽样**可视化：粒子/网格快速翻动 → 收敛 → 边际概率结晶成板。
- **内幕锁定**：官方确定的队-人位（1/2/3/4/6/7）盖下绿色 `LOCK` 钢印。→ 下一步。

### 幕 5 · 最终榜单（FINAL BOARD · 球星卡）
- 30 顺位以**球星卡**形式排布：真实定妆照 + 队徽 + 队色 + 命中置信条 + LOCK 标记。
- 点击任一卡 → **翻牌/放大**成完整球星卡：大头像、六维**属性雷达**、身高/臂展/位置、命中率环、球探评分条。
- 副面板：7 道里程碑答题卡 + 红队质询/可审计统计（终端日志风）。
- 顶部常驻 `▶ 一键演示` 与步骤导航，可回看任意一幕。

## 4. 一分钟视频分镜（≤60s，`?demo=1` 自动播放）

| 时间 | 画面 | 旁白要点 |
|--|--|--|
| 0–5s | 幕0 封面故障字标 + 点击进入 + 白闪 | "这是选秀作战室。" |
| 5–12s | 幕1 启动日志 + HUD 上线 | "30 位 GM、三路信号、上千次推演。" |
| 12–22s | 幕2 天赋 Big Board 级联成形 | "先看谁更强——天赋是根基。" |
| 22–34s | 幕3 市场+资金流入、背离火花、gpt-5.5 裁决 | "再看市场怎么想——分歧就是 alpha。" |
| 34–46s | 幕4 三信号汇合 + 蒙特卡洛收敛 + LOCK 钢印 | "融合、推演上千次、锁定。" |
| 46–58s | 幕5 球星卡铺开 + 点开一张看雷达属性 | "每个顺位，带置信度、可解释、可审计。" |
| 58–60s | 定格在字标 + "提交锁定" | 收尾 |

录制友好：每幕有**明确驻留点**与**自动节拍**；`空格/→` 手动步进；`?demo=1` 走自动时间线。

## 5. 动效语法（映射本机 `03_动效资产/`）

- 幕间转场：`章节炸场_弹出白闪震颤` + `全屏擦除/圆形 Clip 揭示`。
- 入场：`内容级联入场` + `滚动/步骤触发入场`。
- 卡片：悬浮微动 + Persona5 斜切位移；球星卡 = `FLIP 连续物体转场`（列表→大卡）。
- 背景：矩阵数字雨（canvas）；封面更密、内容幕压暗。
- 信号流/管线：SVG 路径流光 dash；蒙特卡洛 = requestAnimationFrame 粒子收敛。
- 全程尊重 `prefers-reduced-motion`：关掉雨/故障/粒子，直接显终态。

## 6. 复用资产（已就位）

- **字体**：得意黑子集 + FusionPixel12（按本页文字子集化）base64 内嵌。
- **定妆照**：`web/headshots/*.png`（ESPN 官方，已压缩；按**球员中文名**映射，base64 内嵌；新板新增 5 人由 agent 补齐）。
- **球队 logo**：NBA 官方 CDN（`cdn.nba.com/logos/nba/{teamId}/…`）+ 失败回退队色徽章。
- **数据**：取 `origin/main` 最新（内幕锁定板 + 统一排序 + 官方答题卡）；`PICKS/MS/AUDIT/RT` 字面量保留以兼容 `scripts/build_frontend.py`。
- **雷达/球星卡**：六维球探评分（得分/投射/组织/篮板/防守/运动）= 按位置+顺位确定性模型估计，页面标注，真实 combine 接入即替换。

## 7. 不要做（`资产总览.md` 红线）

- 不要把结果平铺成静态长页（本次正是要避免）。
- 不要让主色绿覆盖一切；不要低透明正文装高级。
- 不要给无语义变化的装饰加动效；高冲击主角 ≤3 个，其余轻反馈。
- 不要大框套小内容（占位符观感）。

## 8. 交付与部署

- 单文件 `web/draft_room.html`（自包含，含字体/头像 base64）→ EC2 `~/web/index.html`（:8080）。
- `?demo=1` 自动演示；默认手动渐进。
- 兼容 `build_frontend.py`（仅替换 4 个数据字面量块）。
