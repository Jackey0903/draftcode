# 🏀 选秀作战室 · Draft War Room
## 24h 黑客松 — NBA 选秀预测 Agent 系统 · 详细实现文档(终版 v3.0)

> **一句话定位**：我们没有训练一个预测模型,而是重建了一座拥有 **30 位独立人格 GM** 的选秀作战室——它融合**天赋(数据)、专家(mock)、资金(赔率)**三路信号,能自我推演上千次,把"**信号间的背离**"变成超额预测力,并为每一个顺位给出**带置信度、可解释、可审计**的预测。

> 文档版本：**v3.0(三信号融合 · 赔率 Agent 终版)** ｜ 适用赛事：2026 AWS Summit Shanghai · 模拟球探 24h 极限赛
> 评分结构：代码质量 30% / 路演 30% / 里程碑预测 40%
> **v3.0 相对 v2 的改动**：① 新增**赔率聚合 Agent(资金信号)**并融入全链路;② 创新点1 从"双信号"升级为**三信号融合 + 两轴背离**;③ 新增**资金信号处理实现规范**(隐含概率换算 / de-vig 去水 / 线移动 / 信号权威区 / 赔率作蒙特卡洛锚 / 赔率→里程碑);④ 架构图、AWS 映射、执行计划、路演剧本相应更新。
>
> *合规说明:本系统将公开赔率仅作为**预测信号(隐含概率)**使用,不构成、也不提供任何下注建议——这是标准的概率预测数据科学方法。*

## 1. 战略定位:我们在玩的真实游戏

表面叫"有标准答案的黑客松",但**标准答案不是我们赢的地方**。

| 维度 | 权重 | 可控性 | 区分度 | 真相 |
|---|---|---|---|---|
| 里程碑预测 | 40% | 低(含不可消除方差) | 低(前 5 顺位人人猜对) | **陷阱**:看着权重最高,大部分是白送共识 + 运气 |
| 代码质量 | 30% | **100% 可控** | 高 | **主战场**:奖励云原生有多"现代"(Serverless > EC2、容器化、IaC、安全) |
| 路演 | 30% | **100% 可控** | 高 | **主战场**:奖励**方案逻辑合理性**,不是功能堆叠 |

### 1.1 三条不可动摇的设计原则

1. **不为"整张名单全对"优化(概率 ≈ 0),为"每个顺位的期望得分"优化。**
2. **以市场为锚,把天赋当调整项;且市场分两层——专家(mock)+ 资金(赔率)。** 天赋 ≠ 顺位(§1.2)。
3. **60% 精力投在 100% 可控的代码 + 路演上。** 多数队会反过来。

### 1.2 三路信号:这是 v3 的决策核心

| 信号 | 来源 | 本质 | 覆盖 | 在决策里的角色 |
|---|---|---|---|---|
| **天赋信号** | 核心手册(速率指标/高阶/身体) | 数据 | 全部球员 | 根基 / Big Board |
| **专家信号** | ESPN/CBS/Ringer 等 mock | 分析师意见 | 全部 1–30 顺位 | 全榜锚 |
| **资金信号** | 主流博彩 + 预测市场赔率 | 真金白银的隐含概率 | **顶部锐利,中后稀薄** | **顶部主信号 / 最快的情报反映** |

**关键洞察一:天赋 ≠ 顺位(手册自己证明了)。** M.Johnson 天赋 47 分但 ESPN 第 60;C.Anderson 42 分但 ESPN 第 66——典型"盒子数据球员",市场不买账。

**关键洞察二:资金 ≠ 专家,且资金更快。** 当 Shams 报道 Peterson 只给奇才安排试训,**赔率立刻异动,而 mock 还没全更新**。"专家与资金的背离"是一条全新 alpha 轴。

> **战略含义**:绝不能把天赋排序当预测交;也绝不能只信 mock。**三路信号的背离,是我们最该深挖的 alpha。** 这是创新点1的来源。

## 2. 五大创新点(路演 + 代码评分核心)

> 每点遵循:**是什么 → 为什么新颖 → 为什么有效 → 怎么得分**。

### 🥇 创新点 1:三信号融合 + 两轴背离(Tri-Signal Fusion with Two-Axis Divergence)

- **是什么**:对每个球员同时计算**天赋(数据)、专家(mock 加权共识)、资金(赔率 de-vig 后的隐含概率)** 三路信号。三者一致 → 高置信锁定;**背离 → 触发 LLM 深度推理 + 情报破平局**。背离沿**两条轴**展开:
  - **轴① 天赋 vs 市场** → 抓 stats-darling(数据漂亮但市场不认)。
  - **轴② 专家 vs 资金** → 抓"资金知道、分析师还没反应"的内幕(资金通常领先)。
- **为什么新颖**:市面模型几乎都**单信号**——要么纯统计(抓不到市场/钱),要么只复述一份 mock(无原创、无资金视角)。**我们把三路信号的分歧本身当作最强预测信号。**
- **为什么有效**:背离点是预测最难、区分度最大处。资金信号已被实战验证会因内幕快速异动(Peterson/Shams 案例;Dybantsa -370→-600 的漂移)。
- **怎么得分**:路演核心故事("我们把分歧变成 alpha,而且追踪钱的流向");代码上是一个可解释的三信号融合引擎。

### 🥈 创新点 2:30 位独立人格 GM Agent(30 GMs with Decision DNA)

- **是什么**:为 30 队各建一个带独立"决策 DNA"的 GM Agent——知道本队顺位 + 阵容缺口 + 该队 GM 历史选人人格(BPA/补位置、原型偏好、风险容忍、时间线)。
- **为什么新颖**:多数模型把球队抽象成匿名需求向量;**我们建模 30 个有差异的决策者**,还原选秀本来的博弈样貌。
- **为什么有效**:直接补上纯天赋模型的最大盲区——球队层;天然产出可解释选择。
- **怎么得分 + 工程优雅**:30 份人格档案是"绝妙想法 + 实打实工作量";**用 Bedrock 原生多智能体 + Inline Agents 实现**(§4.3)——30 套人格、一份可维护代码。

### 🥉 创新点 3:蒙特卡洛选秀作战室 → 校准的概率预测(Monte Carlo War Room)

- **是什么**:让作战室**自我推演 1000+ 次**,每顺位从概率分布采样;输出众数球员 + 边际概率(= 置信度)。
- **为什么新颖/有效**:① 根除"多米诺级联"(跨千次模拟误差互相抵消);② 每顺位拿到概率分布即诚实置信度;③ 同一批模拟顺手回答所有里程碑客观题(§7.6)。**资金信号还能直接给顶部采样概率当锚**(§7.4)。
- **关键效率设计**:LLM 只跑一次产偏好函数,蒙特卡洛是纯算力采样,1000 次轻松跑完。
- **怎么得分**:置信度可视化 + 校准曲线;**Step Functions 分布式 Map 并行**是顶级云原生范例。

### 🏅 创新点 4:实时情报 Agent + 资金线移动(News Velocity & Line-Movement Edge)

- **是什么**:情报 Agent 在锁定前扫描最新新闻;**赔率 Agent 追踪线移动**——两者共同捕捉最后 48h 的变化,触发重跑。
- **为什么新颖/有效**:选秀最后 48h 由新鲜内幕驱动,静态模型抓不到;**而资金对内幕的反应比 mock 更快(已验证)**。线移动 = 可量化的情报信号。这是 Agent 打败 Model 的根本理由。
- **时机点**:提交 6/24 08:00 ≈ 美东 6/23 20:00 第一轮开始——抓得到几乎全部赛前情报与最新赔率,但选秀本身还没发生。**流水线最后一步 = 情报 + 赔率全新扫描 + 重跑。**

### 🎖 创新点 5:红队自我对抗 + 全程可审计推理(Red-Team & Auditable Reasoning)

- **是什么**:① 魔鬼代言人 Agent 挑战共识(过拟合?群体思维?忽略伤病?);② 每顺位产出"作战室记录"——为什么这么选 + 引用的数据/赔率/情报来源。
- **为什么有效/怎么得分**:红队循环提升鲁棒性;可审计推理满足"答案必须 Agent 跑出、可追溯"红线 + 路演素材 + 工程质量分。**把合规约束变成卖点。**

## 4. AWS 云原生工程(终版重点)

> 评分原文:"看的不只是服务种类的堆叠,而是云原生技术用得有多'现代'、多'合理'";"**容器化、IaC、全链路安全的多维交织,才是识别新手与老炮的分水岭**";"用 Well-Architected 彰显技术格局"。逐条对标。

### 4.1 服务全景映射(全 Serverless + Serverless 容器,零 EC2)

| 层 | AWS 服务 | 说明 |
|---|---|---|
| **采集(容器)** | EventBridge Scheduler + **AWS Fargate**(+ ECR) | 爬虫需 headless 浏览器,用 Serverless 容器(§4.2);**含赔率/预测市场爬取** |
| 对象存储 | S3 | 原始数据、模拟产物、作战室日志 |
| **结构化知识** | **DynamoDB** | 球员/体测/阵容/顺位/**赔率隐含概率**——Agent 确定性查询(数字不进向量库) |
| 非结构化知识 | **Bedrock KB**(+ 可选 OpenSearch Serverless) | scouting 报告、新闻做 RAG(§4.6 有降级路径) |
| 天赋引擎 | Lambda | 手册框架算 Big Board |
| **多智能体** | **Bedrock 多智能体协作(Supervisor + Collaborators)** | Commissioner 编排;球探/专家/**赔率**/情报/红队为子 Agent;30 GM 用 Inline Agents |
| **编排/模拟** | **Step Functions(Distributed Map)** | 序贯选秀 + 1000 次蒙特卡洛大规模并行 |
| 输出服务 | API Gateway + Lambda | 暴露预测 + 作战室记录 |
| **IaC** | **AWS CDK** | 含 Agent 团队的整套系统一键部署 |
| **安全** | IAM 最小权限 + KMS + Secrets Manager + **Bedrock Guardrails** | 全链路(§4.4) |
| 可观测 | CloudWatch + X-Ray + Bedrock 多智能体 trace | 日志/指标/追踪/Agent 交互可视化 |

### 4.2 容器化:Fargate Serverless 爬虫(补齐"多维交织")

**为什么需要容器**：实时情报、mock、**赔率与预测市场**多来自 JS 重渲染站点和社媒,爬取常需 **Playwright / headless Chrome**,超出 Lambda 包体与时长限制——**天然属于容器**。

**怎么调和"Serverless > EC2"与"容器化"**：用 **AWS Fargate = Serverless 容器**——既拿到容器化,又完全不管服务器、按需计费。两个评分勾一次拿满。

**设计**：爬虫 + headless 浏览器打包成镜像推 **ECR**;**EventBridge Scheduler** 定时触发 **Fargate Task** 爬取 → 写 S3 → 触发下游。**锁定前重跑**复用同一任务(带"只抓最新"参数)。备选:**Lambda 容器镜像**(最大 10GB,可塞 headless Chrome)。**推荐 Fargate**(长时爬取与浏览器自动化更稳)。

### 4.3 30 GM 的实现:Bedrock 原生多智能体 + Inline Agents

**为什么用原生多智能体**：Bedrock 已 GA 的多智能体协作提供 **Supervisor + Collaborator** 范式,每个子 Agent 自带工具、Action Groups、知识库与 Guardrails,主管自动规划/路由/整合并提供跨 Agent trace,且原生支持 **CDK / CloudFormation**——既现代又省事。

**拓扑**：Commissioner = Supervisor;Collaborators = 球探·分析、专家聚合、**赔率聚合**、实时情报、红队。

**30 GM 落地 — Inline Agents(关键优雅点)**：不为 30 队各建静态 Agent 资源。用 **Inline Agents(运行时动态调整角色/行为)**:**一个 GM 模板,运行时注入每队"人格档案" → 30 个行为完全独立的 GM**。30 套人格 + 一份可维护代码。

**顺手用上两个降本能力**：**Payload Referencing**(共享 Big Board / 赔率表用引用传递,不每次重复嵌入 → 降本提速);**模型分层**(主管/背离推理用强模型,常规 GM 选人用轻模型)。

**与蒙特卡洛衔接**：多智能体层做"决策建模"产出偏好函数(LLM-once),再交给 Distributed Map 做廉价的 1000 次采样。

### 4.4 全链路安全

**真实风险**：多智能体 + 把**爬来的新闻/赔率页喂进 Agent** = 提示注入攻击面。安全研究(Palo Alto Unit 42, 2026)已证实 Bedrock 多智能体存在此类攻击链;我们的**情报 Agent、赔率 Agent 恰在吃不可信外部内容**。

**逐层对策(真·全链路)**：① 吃外部内容的 Agent(情报/专家/赔率)启用 **Bedrock Guardrails 内置 prompt attack 防护**(AWS 实测可拦);② 每个 Action Group 配独立 **IAM 最小权限角色**;③ S3/DynamoDB **KMS** 静态加密;④ 凭证进 **Secrets Manager**,零硬编码;⑤ TLS 传输加密 + **VPC Endpoint** 私网访问。

### 4.5 Well-Architected 六支柱对照

| 支柱 | 落地 |
|---|---|
| 卓越运营 | CDK(IaC)一键部署 + Bedrock 多智能体 trace + CloudWatch/X-Ray |
| 安全 | IAM 最小权限 + KMS + Secrets Manager + **Guardrails 防注入** + VPC Endpoint |
| 可靠性 | Step Functions 重试容错;DynamoDB 高可用;爬取失败自动回退源 |
| 性能效率 | Distributed Map 并行 + **LLM-once 解耦** + 模型分层 + 容器/函数按负载分配 |
| 成本优化 | 全 Serverless 按需 + Payload Referencing + 轻模型跑常规 GM + 模拟不调 LLM |
| 可持续 | 无闲置资源,事件驱动 |

### 4.6 别过度工程:RAG 降级路径

OpenSearch Serverless 全套 RAG 对 24h 偏重。**留降级路径**:用 Bedrock KB 快速创建,或把有限非结构化内容直接作上下文。**主力是结构化数据(DynamoDB),RAG 是配菜**——别让它吃掉关键路径时间。

### 4.7 开发工作流加分:Kiro 规范驱动

用 **Kiro IDE(规范驱动 Spec-driven)** 落地多智能体系统,把架构决策与测试沉淀为规范文件,利好"AI 为你作证"评分;Agent Hooks 自动跑测试与部署。

## 5. 为什么能赢(逻辑闭环)

一句话:**手册(天赋)只完成一半;我们补上专家(mock)、资金(赔率)、球队层、实时层,用 30 个 GM 在蒙特卡洛作战室里博弈,把三路信号的分歧变成 alpha,为每个顺位给出可解释、带置信度、可审计的预测。**

- 对 **40% 预测**:三信号锚定(不在共识丢分)+ 两轴背离推理(在难点拿分)+ 蒙特卡洛(根除级联、给置信度)+ **赔率直接支撑里程碑题** + 实时情报/线移动(抓最后 48h)。
- 对 **30% 代码**:全 Serverless + 容器化(Fargate)+ Bedrock 原生多智能体/Inline Agents + 分布式 Map + CDK + 全链路安全 + Well-Architected。
- 对 **30% 路演**:无懈可击的故事(三信号 + 追踪钱的流向)+ 置信度可视化 + 每顺位"展示思考"。

## 6. 知识底座(五层)

> 核心手册 = 第 1 层(天赋),已就绪。要赢必须补齐另外四层,尤其直接得分的**球队层**和**实时层**;市场层在 v3 分为**专家(mock)+ 资金(赔率)**两个子源。

## 7. 核心方法论与实现

### 7.1 天赋层 → Big Board(用手册框架)

速率指标为脊(BLK% r≈0.70、REB% r≈0.73、AST% r≈0.72)+ FT%/eFG%/TOV% + 年龄修正(大一×1.3…大四×0.7)+ 位置内 Z-score,BPM 校验 → 位置调整天赋分。

### 7.2 专家层 → mock 加权共识

聚合多家 mock,按各家历史命中率加权 → 每个球员在每个顺位的**专家概率** `MockSignal(p, slot)`。**进阶:对近 48h 更新的 mock 加更高权重**(越新越接近真相)。

### 7.3 三信号融合 + 两轴背离推理(创新点 1 落地)

```
对每个球员 p:
  axis1 = | rank_talent(p) − rank_market(p) |     # 天赋 vs 市场
  axis2 = | implied_mock(p) − implied_odds(p) |    # 专家 vs 资金
  if axis1 小 且 axis2 小:                          # 三方一致 → 高置信锁定
      adjusted(p) = blend(talent, mock, odds)
  else:                                            # 背离 → 触发 LLM 推理 + 情报
      reason = ScoutAgent.analyze(p, talent, mock, odds, live_intel, line_movement)
      # axis1 大: 市场退烧的 stats-darling? 数据没追上的上升期?
      # axis2 大: 资金知道、分析师没反应的内幕? (通常倾向资金, 因其更快/会自我修正)
      adjusted(p) = LLM 修正评估 + 置信度调整
      记录 reason 到作战室日志(可审计)
```

### 7.4 ★资金信号处理(赔率聚合 Agent 实现规范)

这是 v3 的新核心。赔率聚合 Agent 负责把公开赔率转成可融合的概率信号。

**(a) 美式赔率 → 隐含概率**
```
负赔率 -X :  implied = X / (X + 100)        # 例: Dybantsa -550 → 550/650 = 84.6%
正赔率 +X :  implied = 100 / (X + 100)       # 例: Peterson +290 → 100/390 = 25.6%
```

**(b) 去水 / de-vig(必做,否则概率和 > 100%)**
博彩赔率含抽水(overround),同一市场所有候选人的原始隐含概率之和 > 1。用**比例法**归一化:
```
对"#1 顺位"市场的所有候选人 i:
  q_i = 原始隐含概率
  S   = Σ q_i        ( > 1, 即抽水)
  p_i = q_i / S       # 去水后的真实隐含概率, Σ p_i = 1
```
*进阶可选:Shin 法 / power 法,对热门-冷门偏差修正更好;24h 内比例法足够,且作为"我们做了 de-vig"的严谨性亮点。*

**(c) 跨书 / 跨市场共识**
多书取共识:**更锐利的源权重更高**(预测市场如 Polymarket/Kalshi、低抽水书通常更准);或直接用聚合站(Covers/OddsShark)的共识值。预测市场给的是干净的 Yes/No 概率,可直接用。

**(d) 线移动 = 情报信号(喂给情报 Agent)**
追踪赔率随时间漂移(如 Dybantsa 三周 -370→-450→-600;Peterson 在 Shams 报道后异动)。**漂移方向与速度本身是一条可量化的内幕信号**,在锁定前重跑时权重最高。

**(e) 关键洞察:顶部"顺位赔率 = 球队赔率"**
因为顺位顺序固定,"Wilson 第 4 顺位 -750" 等价于 "Wilson 去公牛 -750"。**所以顶部赔率直接给你"哪队选谁 + 概率",无需单独的球队-球员市场。**

**(f) 信号权威区(各信号在哪段最可信)**
| 区域 | 主信号 | 说明 |
|---|---|---|
| **1–4 顺位** | **资金(赔率)** | 盘口锐利流动;顺位赔率=球队赔率;近直接概率 |
| **5–9 顺位** | 资金 + 专家 | 博彩总监原话"赔率从第 5 开始打开";赔率与 mock 融合 |
| **10–30 顺位** | 专家 + 天赋 + 情报 | 赔率稀薄/缺失;mock 覆盖全榜 |
| **里程碑 props** | **资金** | 很多题就是盘口;de-vig 直接给 money-backed 答案(最高 ROI) |
| **球队-球员(顶部外)** | GM Agent + mock | 特定 market 罕见 |

**(g) 赔率作蒙特卡洛锚**:在流动的顶部区(1–4)及任何存在市场的顺位,**直接把采样概率校准到去水后的赔率隐含概率**,而非纯靠偏好函数推导——这把最高杠杆、最高流动的区域锁得最紧。

### 7.5 GM Agent 决策模型(创新点 2,实现见 §4.3)

**每队人格档案(Team Dossier)Schema**(运行时由 Inline Agents 注入):
```yaml
team: "Washington Wizards"
picks: [1, ...]
timeline: "rebuild"
roster_needs: { PG: 0.7, SG: 0.3, SF: 0.5, PF: 0.4, C: 0.8 }
gm_persona:
  philosophy: "BPA-leaning"           # BPA / NEED / 混合
  archetype_pref: ["upside/athleticism", "two-way wing"]
  risk_tolerance: "high"
  intl_openness: 0.6
  notes: "近年偏好年轻上限,愿吞伤病风险"
```

**GM 对剩余球员 p 的偏好函数(v3:含资金项)**:
```
Pref(team, p) =  w_talent  · TalentScore_posadj(p)        # 天赋(数据)
              +  w_need    · NeedFit(team, p)
              +  w_persona · PersonaFit(team, p)
              +  w_expert  · MockSignal(p, team.slot)      # 专家(mock)
              +  w_money   · OddsSignal(p, team.slot)      # 资金(de-vig 赔率)★新
              +  w_intel   · IntelBoost(team, p)           # 含线移动信号

# 权重由 gm_persona + 信号权威区决定:
#   顶部(1-4): w_money 最高(赔率锐利); 中后段: w_money→0, w_expert/talent 升
#   BPA 型: w_talent 高;  NEED 型: w_need 高
```

> **博弈感(写进路演)**:GM Agent 能感知"前面会不会有人截胡我想要的人",边际调整(递归博弈思维)。

### 7.6 蒙特卡洛序贯模拟(创新点 3 + 效率关键)

**核心效率**:LLM 只在 §7.3–7.5 跑一次产偏好函数;蒙特卡洛是纯算力采样,不调 LLM。
```
for sim in 1..N (≈1000):                 # Step Functions 分布式 Map 并行
    available = all_players
    for slot in 1..30:
        team = team_at(slot)             # 处理交易顺位
        if 存在赔率市场(slot):            # 顶部: 直接以去水赔率为采样概率
            probs = devig_odds(slot, available)
        else:
            probs = softmax(Pref(team,p) for p in available, temp=τ(slot))
        # τ 随顺位增大: 前段近确定, 后段高方差 —— 按深度做置信度分层
        pick = sample(available, probs); board[sim][slot]=pick; available.remove(pick)

for slot in 1..30: marginal[slot] = 频率分布(board[*][slot])
final_board = hungarian_assignment(marginal)          # 每人一次, 最大化 Σ 边际概率
confidence[slot] = marginal[slot][final_board[slot]]
```
- **指派优化(Hungarian)**:组合成无重复、期望得分最优的连贯名单。
- **进阶**:前 5 顺位可 LLM-in-the-loop,后段廉价采样。

### 7.7 里程碑客观题(赔率是最高 ROI 来源)

5–8 道客观题(6/23 08:00 上线)**双引擎作答**:
1. **优先查赔率**:很多题本身就是盘口(如"前 5 顺位会否有交易"、"第一个控卫顺位"、"某球员顺位 over/under"),**de-vig 直接给 money-backed 答案**。
2. **蒙特卡洛兜底/校验**:无对应盘口时,在 1000 次模拟结果上算统计量(国际球员平均计数、交易事件占比、首控卫顺位众数)。
3. 两者**交叉验证**,取一致或加权。

> **统一性卖点**:同一套信号体系,赔率出 props 答案、蒙特卡洛出名单与兜底,全部自带置信度。

### 11.3 关键公式

### 11.4 AWS 创新点一句话清单(评委记忆点)
1. **Distributed Map 跑蒙特卡洛** — Serverless 原语并行推演上千次选秀。
2. **LLM-once 解耦** — 昂贵推理只跑一次,采样廉价并行。
3. **Bedrock 原生多智能体 + Inline Agents** — 30 套独立 GM 人格,一份可维护代码。
4. **Fargate 容器 + Lambda/SFN** — 容器与 Serverless 多维交织。
5. **Guardrails 防注入全链路安全** — 指出情报/赔率 Agent 的真实攻击面并给对策。

---

> **核心记忆点**：手册做满了"谁更强";我们补上"专家怎么排、资金怎么押、谁选谁、现在传出什么";用 30 个 GM 在能自我推演上千次的作战室里博弈,把**数据、专家、资金三方的分歧变成 alpha**(而钱是最快的情报),为每个顺位给出**带置信度、可解释、可审计**的预测。架构上,**容器化 + 原生多智能体 + 分布式并行 + 全链路安全 + Well-Architected** 五位一体——这,就是一座能赢、且赢得漂亮的选秀作战室。
