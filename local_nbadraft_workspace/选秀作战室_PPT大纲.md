# 🏀 选秀作战室 · Draft War Room — 路演 PPT 大纲

> **基于**：《选秀作战室·实现文档·终版 v3.0(三信号融合 · 赔率 Agent 终版)》
> **用途**：5–6 分钟现场路演(占总分 30%)+ 兼作评委翻阅的方案说明
> **核心叙事**：我们没有训练一个预测模型,而是重建了一座有 30 位独立人格 GM 的选秀作战室——融合**天赋(数据)、专家(mock)、资金(赔率)**三路信号,自我推演上千次,把"**信号间的背离**"变成超额预测力,为每个顺位给出**带置信度、可解释、可审计**的预测。
> **页数**：10 页(第 1–9 页为路演主线,第 10 页为备份/工程深度,时间紧可略)
> **评分映射**：里程碑预测 40% · 代码质量 30% · 路演 30%

- 主标题:**选秀作战室 · Draft War Room**
- 副标题:24h 黑客松 — NBA 选秀预测 Agent 系统 · 终版 v3.0

**视觉建议**：作战室隐喻视觉(30 个席位 / 大屏);底部一行三个信号标签:天赋 · 专家 · 资金。
**合规小字**：公开赔率仅作**预测信号(隐含概率)**使用,不构成下注建议——标准概率预测数据科学方法。

## 第 2 页 — 我们在玩的真实游戏(~25s)

**核心论点**：40% 的预测分是陷阱;真正可控、可拉开差距的是代码与路演。

| 维度 | 权重 | 可控性 | 真相 |
|---|---|---|---|
| 里程碑预测 | 40% | 低(含不可消除方差) | **陷阱**:看似最高,大半是白送共识 + 运气 |
| 代码质量 | 30% | **100% 可控** | **主战场**:奖励云原生有多"现代" |
| 路演 | 30% | **100% 可控** | **主战场**:奖励方案逻辑合理,而非功能堆叠 |

- 三条设计原则:
  1. 不为"整张名单全对"(概率≈0)优化,为**每个顺位的期望得分**优化。
  2. **以市场为锚,天赋当调整项**;且市场分两层——专家(mock)+ 资金(赔率)。
  3. **60% 精力投在 100% 可控的代码 + 路演上**(多数队反着来)。

**视觉建议**:权重条形图,把"预测 40%"标成陷阱色,代码/路演标成主战场色。

## 第 3 页 — 核心洞察 · 三路信号与两轴背离(痛点+洞察,~75s)

**核心论点**:分歧就是 alpha,而钱是最快的情报。

| 信号 | 来源 | 本质 | 覆盖 | 角色 |
|---|---|---|---|---|
| **天赋** | 核心手册(速率指标/高阶/身体) | 数据 | 全部球员 | 根基 / Big Board |
| **专家** | ESPN/CBS/Ringer 等 mock | 分析师意见 | 全部 1–30 顺位 | 全榜锚 |
| **资金** | 博彩 + 预测市场赔率 | 真金白银隐含概率 | 顶部锐利、中后稀薄 | **顶部主信号 / 最快情报** |

- **洞察一:天赋 ≠ 顺位**(手册自证)。M.Johnson 天赋 47 分却 ESPN 第 60;C.Anderson 42 分却第 66——盒子数据漂亮,市场不买账。
- **洞察二:资金 ≠ 专家,且资金更快**。Shams 报道 Peterson "只见奇才",**赔率立刻异动而 mock 还没全更**;Dybantsa 三周 -370→-450→-600 的漂移。
- **两轴背离**:① 天赋 vs 市场;② 专家 vs 资金。**背离点 = 预测最难、区分度最大处。**

**视觉建议**:两条背离轴的散点/箭头图,高亮 Peterson、Dybantsa 案例。
**金句**:"分歧就是 alpha,而钱是最快的情报。"

## 第 4 页 — 五大创新点总览(方法铺垫,~30s)

**核心论点**:五个创新点,创新点 1 是发动机,其余四个是配套引擎。

- 🥇 **创新点 1 · 三信号融合 + 两轴背离**:把三路信号的分歧本身当作最强预测信号——一致则高置信锁定,背离则触发 LLM 深度推理 + 情报破平局。**(本方案核心)**
- 🥈 **创新点 2 · 30 位独立人格 GM Agent**:为 30 队各建带"决策 DNA"的 GM(顺位 + 缺口 + GM 历史人格),还原选秀博弈本貌。
- 🥉 **创新点 3 · 蒙特卡洛选秀作战室**:自我推演 1000+ 次,每顺位输出众数球员 + 边际概率(= 诚实置信度),根除多米诺级联。
- 🏅 **创新点 4 · 实时情报 + 资金线移动**:锁定前扫描最新新闻 + 追踪线移动,捕捉最后 48h 变化并重跑。
- 🎖 **创新点 5 · 红队自对抗 + 全程可审计推理**:魔鬼代言人挑战共识;每顺位产出"作战室记录",把合规约束变成卖点。

**视觉建议**:五卡片网格,创新点 1 放大/置顶为主卡。

## 第 5 页 — 系统架构 · 多智能体作战室

**核心论点**:Serverless 容器 + Bedrock 原生多智能体 + 分布式并行,一座能并行推演上千次的作战室。

```
【容器化采集层】 EventBridge → AWS Fargate 爬虫(headless)
   球员/历史 · mock · 赔率&预测市场 · 新闻/社媒  → S3
        ▼
【五层知识底座】 结构化 DynamoDB + 非结构化 Bedrock KB / RAG
        ▼
【Bedrock 原生多智能体作战室】  ⟵ Guardrails 防注入
   Commissioner(Supervisor)— 规划/路由/整合/trace
     ├ 球探·分析 → 天赋 Big Board(数据)
     ├ 专家聚合 → mock 加权共识(专家)
     ├ 赔率聚合 ★新 → de-vig 隐含概率 + 线移动(资金)
     ├ 实时情报 → News Velocity(吃外部内容,重点防注入)
     ├ 30× GM(Inline Agents 注入人格档案)
     └ 红队 → 自我对抗修正
   ↓ 三信号融合 + 两轴背离 → 每队"偏好函数"
        ▼
【蒙特卡洛引擎】 Step Functions Distributed Map ×1000;顶部以赔率为锚
   → 边际概率 → 匈牙利指派 → 一致且期望最优名单 + 置信度
        ▼
【交付】 API Gateway + Lambda:30 顺位+置信度 · 里程碑答案 · 作战室记录
横切:CDK(IaC) · CloudWatch/X-Ray · IAM 最小权限 + KMS + Secrets Manager + Guardrails
```

**架构叙事(口播)**:"我们用 Serverless 容器 + 原生多智能体 + 分布式并行,重建了一座融合数据、专家、资金的作战室——能并行推演上千次,**还追踪钱的流向**,对每个预测敢标置信度。"

## 第 6 页 — AWS 云原生工程(技术深度,~60s)

**核心论点**:容器化 + 原生多智能体 + 分布式并行 + 全链路安全多维交织——这是识别新手与老炮的分水岭。

- **Fargate = Serverless 容器**:爬虫需 Playwright/headless Chrome,超出 Lambda;用 Fargate 同时勾满"Serverless > EC2"与"容器化",零 EC2。
- **Bedrock 原生多智能体**:Commissioner = Supervisor,球探/专家/**赔率**/情报/红队 = Collaborators,自带 Action Groups/KB/Guardrails + 跨 Agent trace。
- **Inline Agents(优雅点)**:不为 30 队各建静态资源——**一个 GM 模板,运行时注入人格档案 → 30 个行为独立的 GM,一份可维护代码。** 配 Payload Referencing + 模型分层降本。
- **Step Functions Distributed Map**:1000 次蒙特卡洛大规模并行;**LLM-once 解耦**——昂贵推理只跑一次,采样廉价并行。
- **全链路安全**:吃外部内容的 Agent 启用 **Guardrails 防注入**(真实攻击面,Unit 42 已证实) + Action Group 独立 IAM 最小权限 + KMS + Secrets Manager + VPC Endpoint。
- **Well-Architected 六支柱** + **CDK 一键部署** + Kiro 规范驱动。

**视觉建议**:左侧服务图标墙,右侧四个"AWS 创新点"高亮条。

## 第 7 页 — 方法论闭环(方法主干,~60s)

**核心论点**:同一套信号体系,从原始数据一路推到带置信度的名单与里程碑答案。

1. **天赋层 → Big Board**:速率指标为脊(REB% r≈0.73、AST% r≈0.72、BLK% r≈0.70)+ FT%/eFG% + 年龄修正 + 位置 Z-score。
2. **专家层 → mock 加权共识**:按各家历史命中率加权;近 48h 更新加更高权重。
3. **资金层 → 赔率聚合 Agent**(v3 新核心):
   - 美式赔率 → 隐含概率(-550 → 84.6%;+290 → 25.6%)
   - **de-vig 去水**(比例法 `p_i = q_i / Σq_j`,严谨性亮点)
   - 跨书/预测市场共识 · **线移动 = 可量化情报**
   - 关键:**顶部"顺位赔率 = 球队赔率"**,直接给"哪队选谁 + 概率"
4. **三信号融合 + 两轴背离**:一致 → blend 锁定;背离 → LLM 修正 + 记录可审计 reason。
5. **30 GM 偏好函数**:`Pref = w_talent·Talent + w_need·Need + w_persona·Persona + w_expert·Mock + w_money·Odds + w_intel·Intel`(权重随**信号权威区**变化:1–4 顺位资金最高,中后段转专家/天赋)。
6. **蒙特卡洛 + 匈牙利指派**:每顺位边际概率 → 无重复、期望最优名单;`confidence = marginal[slot][pick]`。
7. **里程碑客观题**:**赔率优先 de-vig 直答(最高 ROI)+ 蒙特卡洛兜底/交叉验证。**

**视觉建议**:横向流程条(7 步),de-vig 与蒙特卡洛各配一个小公式徽标。

## 第 8 页 — 现场演示 · 让作战室自己说话(Demo,~75s)

**核心论点**:挑一个两轴都背离的顺位,展示"展示思考"的能力。

- **演示 1 — 作战室记录**:选一个天赋 vs 市场、专家 vs 资金**双背离**的顺位,逐条展示:数据说什么、专家排第几、资金怎么押、情报/线移动如何破平局,以及最终为什么这么定。
- **演示 2 — 置信度可视化**:展示每顺位边际概率分布 / 校准曲线——前段近确定、后段高方差,诚实分层。
- **演示 3 — 赔率秒答里程碑**:用 de-vig 后的盘口**直接秒答一道客观题**(如"第一个控卫顺位""某球员顺位 over/under"),money-backed。

**视觉建议**:三联屏——左作战室记录卡、中置信度分布图、右里程碑答题卡。
**口播提示**:强调"答案是 Agent 跑出来的、可追溯",呼应可审计红线。

## 第 9 页 — 为什么能赢(逻辑闭环 + 收尾,~30s)

**核心论点**:对三个评分维度各有打法,且故事无懈可击。

- **对 40% 预测**:三信号锚定(不在共识丢分)+ 两轴背离(在难点拿分)+ 蒙特卡洛(根除级联、给置信度)+ **赔率直接支撑里程碑**+ 线移动抓最后 48h。
- **对 30% 代码**:全 Serverless + Fargate 容器 + 原生多智能体/Inline Agents + Distributed Map + CDK + 全链路安全 + Well-Architected。
- **对 30% 路演**:三信号 + 追踪钱的流向的故事 + 置信度可视化 + 每顺位"展示思考"。
- **收尾金句(口播)**:
  > "我们不追求'整张名单全对'——那概率约等于零。我们为每个顺位的期望得分优化,融合数据、专家、资金三方,并对每个预测敢标置信度、敢展示思考。**这就是一支能解释自己的作战室。**"

**视觉建议**:三栏"评分维度 → 打法"对照 + 居中收尾金句。

## 第 10 页 — (备份/工程深度)24h 执行计划与风险对冲

**核心论点**:不只想得清,更跑得出来——展示工程纪律。时间紧可略过或留作 Q&A。

- **逐时段精简**:09:00 立地基 → 知识底座(含资金层)→ 三引擎(天赋/专家/赔率融合)→ 30 GM + 编排 → 蒙特卡洛作战室 → 红队 + 客观题 → 打磨(安全收口/PPT/视频)→ **🔴 06:00–07:30 锁定前重跑(情报+赔率全新扫描→重跑→最终名单)** → 提交。
- **关键容错(降级路径)**:
  - 赔率 geo 受限 → 抓媒体/聚合站转载赔率 + 爬虫跑云上 + 手动兜底。
  - 蒙特卡洛慢 → 先本地 200 次出 v1,云上并行增强。
  - Fargate 卡 → 降级 Lambda 容器镜像。
  - 30 GM 来不及 → 通用人格模板 + 先做前 14 顺位精细人格。
  - 多智能体编排不熟 → 单 Agent + Lambda 顺序调度保主流程。
- **交付物**:Private GitHub(CDK + README + 作战室记录 + Kiro Specs)· 答题卡(30 顺位+置信度+5–8 客观题)· PPT + ≤60s 视频。

**视觉建议**:24h 时间轴甘特条,把"锁定前重跑"标成红色关键节点。
