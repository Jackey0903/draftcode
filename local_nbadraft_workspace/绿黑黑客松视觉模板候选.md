# 绿黑黑客松视觉模板候选

更新时间: 2026-06-23

用途: 给 `ReDraft / Draft War Room` 黑客松路演、展示页、控制台原型挑选视觉方向。

## 我读完本地设计资产后的判断

这个项目不适合直接套“黑底 + 霓虹绿 + 网格 + 到处发光”的普通黑客模板。你的 `设计元素` 资料库强调的是:

- 先定具体视觉世界,再写 CSS。
- 主色必须有职责,不能把绿色贴满所有边框、标题和按钮。
- 一屏一个主版式,动效服务叙事、状态变化或操作反馈。
- 首屏要像开场承诺,不是普通内容页放大标题。
- 高冲击效果最多 1-3 个主角,其余用轻反馈和节奏控制。

所以绿黑风格要被改造成“高级作战室 / 数据战情 / 选秀夜信号系统”,而不是廉价 hacker 背景。

## 外部素材筛选原则

- 能直接落地: 有代码、模板、框架、PPT 或演示骨架。
- 能改成绿黑: 原始素材不必就是绿黑,但结构要适合换肤。
- 适合 24h 黑客松: 可快速拼成路演、控制台或动效 demo。
- 和你的资产库兼容: 能接入 `WebGL噪点首屏`、`点阵数字`、`玻璃控制台系统`、`内容级联入场`、`流程路线`、`章节冲击页` 等。
- 不牺牲演示稳定性: WebGL/粒子/blur 都要有降级版本。

## 候选总览

| 编号 | 模板方向 | 最适合做什么 | 外部源类型 | 落地难度 | 我的建议 |
|---|---|---|---|---:|---|
| T01 | Phosphor War Room | 最终路演主视觉 + dashboard | Arwes / TailAdmin / Black Dashboard | 中 | 最推荐 |
| T02 | Matrix Draft Night | 首屏炸场 + 章节转场 | Matrix WebGL / CMatrix | 中 | 可做强记忆点 |
| T03 | Terminal Commissioner | 评委现场演示 / Agent 可审计日志 | Terminal portfolio / Reveal.js | 低 | 很适合黑客松 |
| T04 | Cyberdeck Control Console | 交互原型 / 30 GM 作战室 | Arwes / Cybercore CSS | 中高 | 最像“系统” |
| T05 | WebGL Signal Field | 高级封面 / 背景材质场 | Vanta / GSAP / Motion | 中 | 气质高级但要克制 |
| T06 | Reveal Green Room Deck | 纯路演幻灯片 | Reveal.js / reveal-hugo / PPT 模板 | 低 | 最稳 |
| T07 | Hackathon Command Site | 项目官网 / 提交页 | Hack Club website refs | 低中 | 适合补展示入口 |
| T08 | Minimal Quant Terminal | 轻量单文件 HTML | anime.js / Motion / CSS | 低 | 最适合 24h 兜底 |

## T01 Phosphor War Room

一句话: 把 `Draft War Room` 做成一间低亮度的球队数据作战室,绿色只代表“系统就绪 / 置信通过 / 信号锁定”。

### 适合

- 最终路演主版本。
- 30 GM Agent / Monte Carlo / 置信度可视化。
- 需要让评委觉得“这套系统真的能跑”的展示。

### 参考源

- [Arwes](https://arwes.dev/docs): 科幻 UI 框架,适合拿面板、线框、进入动效和 sci-fi 控件语法。
- [TailAdmin React](https://github.com/TailAdmin/free-react-tailwind-admin-dashboard): React + Tailwind dashboard 骨架,当前 GitHub API 查到 MIT,约 1.2k stars。
- [Black Dashboard React](https://github.com/creativetimofficial/black-dashboard-react): 暗色 dashboard 结构,MIT,适合换成绿黑数据面板。
- [Mosaic Lite](https://github.com/cruip/tailwind-dashboard-template): React + Tailwind dashboard,图表和 widgets 很适合作战室。

### 模板骨架

受众/场景: NBA 数据、AI、工程评委;6 分钟黑客松路演。

视觉世界: 选秀夜后台作战室,像球队总经理正在盯一块低亮度战情屏。

拒绝套路: 不用满屏 Matrix 雨;不用所有卡片绿色描边;不用假装很复杂的大空框。

首屏承诺: 左侧是 `DRAFT WAR ROOM` 超大标题,右侧是 30 队纵向 draft board,少量 pick 卡片出现上移/下调轨迹。

主色/字体/材质:

- 背景: `#030805` / `#07110b`
- 主文字: 冷白 `#eef8f1`
- 绿色: `#39ff88`,只给 ready、confidence、locked
- 辅助色: 琥珀 `#e9c46a` 给风险/uncertainty
- 材质: 黑玻璃面板 + 1px 内高光 + 低强度噪点

页面家族:

- Cover: 选秀夜作战室
- Signal Split: Talent vs Market 双信号
- 30 GM Grid: 30 个 GM agent 小面板
- Monte Carlo Board: 概率热力选秀板
- Audit Log: 每个 pick 的作战室记录
- Architecture: Serverless / Agent / Simulation 三层图

动效语法:

- 内容级联入场: 标题 -> 状态 chip -> draft board。
- FLIP: 球员卡从市场顺位移动到模型顺位。
- 点阵数字: 置信度、模拟次数、pick number。
- 卡片悬浮微动: GM 面板 hover。
- 全屏 wipe: 从普通 deck 切入“live war room”模式。

适配你本地资产:

- `暗色发光展示系统`
- `玻璃控制台系统`
- `流程路线`
- `点阵数字`
- `固定大屏画布`
- `三栏工作区`

风险:

- 容易变成普通 dashboard,需要第一屏和关键页做出“选秀夜”识别物。
- 绿色不能太多,否则廉价。

## T02 Matrix Draft Night

一句话: 用 Matrix digital rain 做“数据从历史选秀夜流回模型”的仪式感,但只放在首屏/章节,不放满全场。

### 适合

- 开场 10 秒强记忆点。
- 历史回测模块: 模型回到过去,只看当时数据。
- 章节冲击页。

### 参考源

- [Rezmason/matrix](https://github.com/Rezmason/matrix): WebGL / REGL 版本 Matrix rain,MIT,GitHub API 查到约 3.8k stars。
- [jcubic/cmatrix](https://github.com/jcubic/cmatrix): Canvas Matrix 效果,MIT,适合轻量嵌入。
- [GitHub matrix-rain topic](https://github.com/topics/matrix-rain): 可继续找更多变体。

### 模板骨架

受众/场景: 路演开头和章节转折。

视觉世界: 不是黑客入侵,而是“历史选秀数据流正在被重放”。

拒绝套路: 不把代码雨铺满全部页面;不使用假的密码/入侵文案;不让正文压在高对比字符雨上。

首屏承诺: 黑色舞台中央出现 `ReDraft`,背后是非常稀疏的绿色字符流;字符偶尔组成 pick number、年份、team code。

主色/字体/材质:

- 背景: `#000604`
- 主绿: `#00ff66`
- 暗绿: `#0a2a18`
- 文字: `#f2fff7`
- 字体: Display 用硬朗几何字体,Mono 只放状态和代码感标签

页面家族:

- Matrix Cover
- Year Rewind: 2016 / 2017 / 2018 年份像终端命令一样切换
- Pick Delta: 上升/下调轨迹
- Data Gate: 从字符雨 wipe 到真实 dashboard

动效语法:

- WebGL/Canvas 字符雨慢速流动。
- 像素显影: 年份和 pick 卡从噪点中显影。
- 章节炸场: 只在“模型回到选秀夜”使用一次。
- 降级: 静态噪点 + 低密度 CSS text grid。

适配你本地资产:

- `WebGL噪点首屏`
- `章节冲击页`
- `像素显影`
- `全屏擦除Wipe`

风险:

- Matrix 容易显得过时,必须改成“数据重放”而不是“黑客入侵”。
- 只适合少数高冲击页面,不适合整套 deck。

## T03 Terminal Commissioner

一句话: 把 Commissioner Agent、红队、30 GM 的推理过程做成可审计终端,黑客松现场很容易讲清楚。

### 适合

- 现场演示 Agent 运行过程。
- 展示“每个顺位为什么这么选”。
- 需要快速落地的单文件 HTML 或 React 页面。

### 参考源

- [satnaing/terminal-portfolio](https://github.com/satnaing/terminal-portfolio): React + TypeScript 终端风格 portfolio,MIT,GitHub API 查到约 780 stars。
- [PythonToGo/terminal-portfolio-template](https://github.com/PythonToGo/terminal-portfolio-template): 轻量 HTML/CSS/JS 终端主页模板。
- [Reveal.js](https://github.com/hakimel/reveal.js): HTML presentation framework,MIT,GitHub API 查到约 71k stars。

### 模板骨架

受众/场景: 评委问“Agent 到底做了什么”时展示。

视觉世界: 一台 Commissioner 控制台正在调度 scouts、market、GM、red-team 子进程。

拒绝套路: 不用假进度条灌水;不让终端长日志占满 8 页;不把所有解释都写成代码。

首屏承诺: 一个极简终端窗口显示:

```text
commissioner@draft-war-room:~$ run simulation --trials 1000 --mode lock
scout.signal      READY
market.signal     READY
gm.agents[30]     READY
red.team          ARMED
```

主色/字体/材质:

- 背景: `#010403`
- 终端绿: `#b6ffcf`
- 系统绿: `#36f58b`
- 警告黄: `#ffd166`
- 风险红: `#ff4d6d`
- 材质: CRT scanline 很轻,不要强烈闪烁

页面家族:

- Command Cover
- Agent Boot Sequence
- Tool Trace
- Pick Audit Log
- Red Team Challenge
- Final Submission Receipt

动效语法:

- 内容级联入场模拟日志逐行出现。
- 光标 blink 只在命令行。
- FLIP: 日志里的 player id 变成右侧 pick card。
- 按压下沉 + 触摸涟漪: 可点命令、tabs、filters。

适配你本地资产:

- `点阵数字`
- `顶部状态Chip`
- `反馈闭环`
- `内容级联入场`
- `交互原型嵌入`

风险:

- 终端风格过多会像开发者玩具,需要搭配清晰图表页。
- 中文正文不适合全部使用 mono。

## T04 Cyberdeck Control Console

一句话: 像一台可以操作的 cyberdeck,中间是选秀板,左侧队伍,右侧球员/置信/理由,适合交互原型。

### 适合

- 可点的作战室 demo。
- 展示 30 GM 独立人格。
- 展示三栏工作区与状态 chip。

### 参考源

- [Arwes GitHub](https://github.com/arwes/arwes): MIT,TypeScript,GitHub API 查到约 7.5k stars。
- [Cybercore CSS](https://github.com/sebyx07/cybercore-css): 纯 CSS cyberpunk framework,MIT,适合借鉴按钮、卡片、边框语言。
- [Arwes docs](https://arwes.dev/docs): 明确面向 futuristic science fiction UI、animations、sound effects。

### 模板骨架

受众/场景: 黑客松 demo booth 或答辩时可互动展示。

视觉世界: 一台可操作的 NBA draft cyberdeck,不是网页,更像任务台。

拒绝套路: 不做装饰性不可点按钮;不放太多入口;不让右栏空着。

首屏承诺: 16:9 固定大屏画布,顶部状态栏显示 `LIVE INTEL / MARKET / TALENT / SIMULATION`,中间是 pick board,底部 dock 切换模式。

主色/字体/材质:

- 背景: `#020706`
- 面板: `#07140d`
- 边框: `rgba(65,255,143,.22)`
- Active: `#39ff88`
- Risk: `#ff5c7a`
- Material: 更硬朗的金属黑玻璃,少柔光

页面家族:

- War Room Home
- Team Detail
- Player Compare
- Simulation Heatmap
- Red Team Overlay
- Submission Lock

动效语法:

- Dock 磁性吸附。
- 圆形 Clip 揭示: 从底部 Dock 打开 team/player/detail。
- 场景接管层: `SUBMISSION LOCKED` 或 `RED TEAM CHALLENGE`。
- 触摸涟漪 + 按压下沉: 所有可点控件。

适配你本地资产:

- `固定大屏画布`
- `顶部状态Chip`
- `底部Dock`
- `三层信息架构`
- `三栏工作区`
- `反馈闭环`

风险:

- 最像产品 demo,但做多了会占用路演制作时间。
- 如果没有真实交互,会露怯;所有按钮必须至少打开菜单、改状态或 toast。

## T05 WebGL Signal Field

一句话: 用 WebGL/Shader 背景做高级信号场,让首屏和关键章节有“活的材质”,而内容仍然克制清楚。

### 适合

- 高级封面。
- “数据与市场背离”这种抽象概念页。
- 需要很强第一眼质感的展示页。

### 参考源

- [Vanta.js GitHub](https://github.com/tengbao/vanta): Animated 3D backgrounds,MIT,GitHub API 查到约 6.5k stars。
- [Vanta 官网](https://www.vantajs.com/): 可配置颜色、鼠标/触摸交互,用 three.js 或 p5.js 渲染。
- [GSAP](https://github.com/greensock/gsap): 高性能动画库,适合 timeline 和 ScrollTrigger。
- [Motion](https://motion.dev/): React/JS/Vue 动画库,适合 UI 入场、layout transition。

### 模板骨架

受众/场景: 第一屏、章节页、重点观点页。

视觉世界: 选秀市场像一个不断扰动的信号场,模型从噪声里提取 alpha。

拒绝套路: 不用粒子宇宙;不让背景抢正文;不在每屏都开 WebGL。

首屏承诺: 暗绿色 fBm 噪点场缓慢流动,鼠标扰动形成局部亮区,右侧 draft cards 被信号扫过。

主色/字体/材质:

- 背景: `#010806`
- 信号绿: `#2cff8f`
- 深青: `#0c5240`
- 冷白: `#f4fff8`
- 材质: 细噪点、低亮度径向光、少量线框

页面家族:

- Signal Field Cover
- Divergence Map
- Confidence Calibration
- Monte Carlo Density
- Live Intel Pulse

动效语法:

- WebGL 噪点首屏。
- 滚动触发入场。
- 连接线绘出。
- 降级: CSS radial gradient + SVG noise texture。

适配你本地资产:

- `WebGL噪点首屏`
- `Aurora背景漂移`
- `流程路线`
- `滚动触发入场`
- `性能与降级`

风险:

- 质感高级但容易空,必须搭配真实数据结构。
- WebGL 要限制 DPR,低端机器降级。

## T06 Reveal Green Room Deck

一句话: 最稳的路演交付路线,用 Reveal.js 做一套绿黑版 8 页 deck,每页一个主版式。

### 适合

- 24h 黑客松最后提交。
- 需要 PDF / HTML 双输出。
- 需要控制讲述节奏。

### 参考源

- [Reveal.js](https://github.com/hakimel/reveal.js): HTML presentation framework,MIT,成熟稳定。
- [reveal-hugo](https://github.com/joshed-io/reveal-hugo): Hugo + Reveal.js 演示稿骨架,MIT。
- [Slidesgo IT Security Hacker Pitch Deck](https://slidesgo.com/theme/it-security-hacker-pitch-deck): 可参考 PPT 章节结构和图形密度。
- [SlidesCarnival cyber templates](https://www.slidescarnival.com/tag/cyber): 可参考 PowerPoint/Google Slides 的 cyber presentation 信息结构。

### 模板骨架

受众/场景: 6 分钟路演。

视觉世界: 绿色战情室里的短篇 pitch film。

拒绝套路: 不套现成 PPT 的锁头、盾牌、网络线;不把每页做成三卡片模板。

首屏承诺: `ReDraft` 作为字标,一条从 `market consensus` 到 `true value` 的绿色信号线穿过 draft board。

主色/字体/材质:

- 背景: 深黑绿
- Accent: 单一荧绿 + 少量金色
- 图表: 绿色表示信号通过,红/琥珀表示背离和风险

页面家族:

1. Cover
2. Blind Spot
3. Dual Signal
4. 30 GM War Room
5. Monte Carlo
6. Live Intel
7. Architecture
8. Submission

动效语法:

- 每页内容级联入场。
- 页间尽量不强 scroll-jacking。
- 章节页可用一次白闪震颤。
- Reduced motion 时关闭大幅动画。

适配你本地资产:

- `首屏封面`
- `版式密度`
- `章节冲击页`
- `命题强调块`
- `视觉QA`

风险:

- 容易太像普通演示稿,需要首屏和 1-2 个关键页有独特视觉对象。

## T08 Minimal Quant Terminal

一句话: 一个最小、稳、能双击打开的单文件 HTML,绿黑量化终端 + 少量高级动效。

### 适合

- 时间不够时的兜底方案。
- 不想引入构建工具。
- 当前 `index.html` 已经是单文件 deck,可直接换肤或抽取一版。

### 参考源

- [Anime.js](https://animejs.com/): 轻量 JS animation engine,适合数字、SVG、DOM 动效。
- [Motion](https://motion.dev/): 适合 React/JS UI 动效;若不用构建,可只借鉴动效语法。
- [Lenis](https://github.com/darkroomengineering/lenis): smooth scroll,但本项目如果保持 CSS scroll snap,可不引入。
- [tsParticles presets](https://github.com/tsparticles/presets): 粒子 preset 可参考,但建议谨慎使用。

### 模板骨架

受众/场景: 最稳可交付演示。

视觉世界: 量化交易终端 + NBA draft board 的混合体。

拒绝套路: 不依赖外网;不引入大包;不做复杂 3D;不让动画影响阅读。

首屏承诺: 只有一个超大 `ReDraft`、一条绿色 signal line、右侧 1-30 pick list 抽象轨迹。

主色/字体/材质:

- 黑: `#000403`
- 绿: `#37ff8b`
- 灰白: `#eaf7ee`
- 辅助黄: `#e6c66a`

页面家族:

- Cover
- Problem
- Method
- War Room
- Simulation
- Audit
- Architecture
- Lock

动效语法:

- CSS transition 为主。
- 数字滚动和路径绘制用小 JS。
- 所有 heavy effect 都可关闭。

适配你本地资产:

- `内容级联入场`
- `流程路线`
- `点阵数字`
- `视觉QA`
- `性能与降级`

风险:

- 视觉惊艳度低于 T01/T05,但稳定性最高。

## 我建议的选择方式

如果你要我直接继续做,我建议三选一:

1. 选 `T01 Phosphor War Room`: 最适合这个 NBA 选秀作战室项目,高级、可信、可扩展。
2. 选 `T01 + T02`: 主体用作战室,开场/章节用 Matrix data rewind,记忆点最强。
3. 选 `T06 + T03`: 最适合 24h 黑客松交付,路演稳,现场解释 Agent 也清楚。
