# 故障发光字标

## 适合

黑客松路演封面、系统启动页、项目名称登场、章节标题、需要在 3 秒内建立“技术感 + 记忆点”的主视觉。

## 效果

超大字标使用高对比白字,叠加绿色外发光、青红轻微错位阴影和短促 glitch 抖动。观众会感觉标题像从数据流里被系统锁定出来,但主体仍然清楚可读。

## 实现抓手

```css
.wordmark {
  color: #f4fff8;
  text-shadow:
    0 0 22px rgba(0, 255, 102, .58),
    2px 0 rgba(255, 80, 120, .7),
    -2px 0 rgba(98, 242, 255, .48);
  animation: glitch-word 2.6s steps(2) infinite;
}

@keyframes glitch-word {
  0%, 86%, 100% { transform: none; filter: none; }
  88% { transform: translateX(-3px); filter: hue-rotate(18deg); }
  90% { transform: translateX(3px); }
  92% { transform: translateX(-1px); }
}
```

## 使用方式

- 只给主项目名、章节短词或关键命题使用。
- 字体要粗、宽、可读,不要用于长句。
- 绿色发光负责“系统锁定/数据激活”,红青错位只做瞬间故障感。
- 抖动频率要低,大部分时间保持稳定。

## 可搭配

- `Matrix数据雨背景`
- `像素显影`
- `章节炸场_弹出白闪震颤`
- `内容级联入场`

## 禁忌

- 不要给正文、列表、按钮使用。
- 不要持续高频抖动,会廉价且影响阅读。
- 不要同时叠太多颜色; 绿是主角,青红只是短促错位。
- 不要让字标超过容器导致裁切,宁可降字号也要完整显示。

## 验收

缩成缩略图仍能认出项目名; 正常观看时先看到清晰标题,其次才感受到发光和故障抖动。
